# AWS PrivateLink

## 개요
- AWS PrivateLink는 **VPC 간 비공개 서비스 연결**을 제공하는 기능.
- 서비스 공급자(Provider)가 자신의 VPC에서 서비스(VPC Endpoint Service)를 노출.
- 서비스 소비자(Consumer)는 자신의 VPC에서 인터페이스 엔드포인트(ENI)를 생성해 연결.
- 모든 트래픽은 AWS 네트워크 내부에서 처리 → 인터넷 불필요.

---

## 특징
- **보안**: 트래픽이 AWS 백본 네트워크를 통해 전송 → 공개 인터넷 사용 안 함.
- **제한적 노출**: 서비스 제공자가 지정한 서비스만 노출 가능.
- **간단한 연결**: 복잡한 라우팅 불필요.

---

## PrivateLink와 다른 옵션 비교
| 옵션              | 특징                                                     | 적합한 경우 |
|-------------------|----------------------------------------------------------|-------------|
| VPC 피어링        | 두 VPC 전체를 연결, 리소스 간 라우팅 필요                 | 같은 계정/리전의 두 VPC 전체 연결 |
| VPN / DirectConnect | 온프레미스 ↔ AWS VPC 연결                               | 온프레미스 ↔ AWS 환경 |
| NAT Gateway       | 프라이빗 서브넷 → 인터넷으로 나가는 트래픽                | 인터넷 접근 필요 시 |
| PrivateLink (정답) | 특정 서비스만 비공개 연결, AWS 네트워크 내 전송           | 외부 공급자 서비스 접근 |

---

## 시험 포인트
- "외부 공급자의 VPC 서비스에 비공개로 연결" → **AWS PrivateLink**  
- "온프레미스 ↔ AWS 연결" → **VPN/Direct Connect**  
- "VPC 전체 간 연결" → **VPC Peering**  
- "프라이빗 → 인터넷" → **NAT Gateway**
